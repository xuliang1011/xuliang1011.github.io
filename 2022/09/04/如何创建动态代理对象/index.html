如何创建动态代理对象

什么是代理

从字面意思来看，代理比较好理解，无非就是代为处理的意思，也就是一个增强方法的工具类。

public interface Service {
    void doService();
    void doOther();
}



public class ServiceImpl implements Service {
    @Override
    public void doService() {
        System.out.println("执行业务方法doService");
    }

    @Override
    public void doOther() {
        System.out.println("执行业务方法doOther");
    }
}


public class Text {
    public static void main(String[] args) {
        //调用doService，doOther
        Service service=new ServiceImpl();
        service.doService();
        System.out.println("---------------------");
        service.doOther();
    }
}

我举一个很简单的一个例子，上面就是我们常用写一个方法的方式，要是这个方法我们需要给它加功能比如加个打印日志和事务，我们应该怎么做呢？最直观就是我们直接在Service方法里面加。

public void doService() {
    System.out.println("方法的执行时间："+new Date());
    System.out.println("执行业务方法doService");

    //方法最后提交事务
    System.out.println("方法执行完毕提交事务。");
}

但这样的话我们就改变了原先代码的程序逻辑。我们所加的功能和原先代码的业务功能是无关的。所以doService方法就会显得特别乱。当然我们也可以这样做，写一个工具类去实现上面的两个那功能。

public class ServiceTools {
    public static void doLog(){
        System.out.println("方法的执行时间："+new Date());
    }
    public static void doTrans(){
        System.out.println("方法执行完毕提交事务。"); 
    }
}


public class ServiceImpl implements Service {
    @Override
       public void doService() {
//        System.out.println("方法的执行时间："+new Date());
        ServiceTools.doLog();
        System.out.println("执行业务方法doService");

        //方法最后提交事务
//        System.out.println("方法执行完毕提交事务。");
        ServiceTools.doTrans();
    }
}

它比之前就简洁了一些，只有方法的调用了，但是还是不够简洁，修改也不方便，以及很多重复的代码。

而我们想要做的更好，在原有基础上，不修改ServiceImpl类里面的任何代码，让它也能去增加输出日志和事务这两个功能。所以这就要用到我们的动态代理了。

那么动态代理怎么使用了

划重点，我理解的动态代理，它就是一个工具类，只要你适应它的规则自然就轻车熟路了。

jdk动态代理实现步骤:
1.创建目标类，SomeServiceImpl目标类，给它的doService，doother增加输出时间，事务。

2.创建InvocationHandler接口的实现类，在这个类实现给目标方法增加功能。

3.使用jdk中类proxy ，创建代理对象。实现创建对象的能力。

现在我们进行第二步：

public class MyInvocationHandler implements InvocationHandler {
    //目标对象
        private Object target;//即我们得ServiceImpl类
        public MyInvocationHandler(Object target){
            this.target=target;
        }
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        //通过代理对象执行方法时，会调用执行这个invoke ()
        Object invoke=null;
        ServiceTools.doLog();
        //执行目标类方法，通过Method类实现
        invoke = method.invoke(target, args);
        ServiceTools.doTrans();
        return invoke;
    }
}

第三步：

public class Text {
    public static void main(String[] args) {
//        调用doService，doOther
//        Service service=new ServiceImpl();
//        service.doService();
//        System.out.println("---------------------");
//        service.doOther();
        //使用jdk的Proxy创建代理对象
        // 创建目标对象
        Service target=new ServiceImpl();
        //创建InvocationHandler对象
        InvocationHandler invocationHandler=new MyInvocationHandler(target);
        //使用Proxy创建代理
        Service proxy = (Service) Proxy.newProxyInstance(
                target.getClass().getClassLoader(),//第一个参数固定的，目标类得类加载器，可以用反射机制获取
                target.getClass().getInterfaces(),//第二个参数，目标类实现了什么接口
                invocationHandler);//第三个参数，代理对象要执行的功能
        //通过代理执行方法，会调用handler中的invoke ( )
        proxy.doOther();
    }
}
invoke方法的第一个参数proxy到底有什么作用？

这个问题其实也好理解，如果你的接口中有方法需要返回自身，如果在invoke中没有传入这个参数，将导致实例无法正常返回。在这种场景中，proxy的用途就表现出来了。简单来说，这其实就是最近非常火的链式编程的一种应用实现。

动态代理到底有什么用？

学习任何一门技术，一定要问一问自己，这到底有什么用。其实，在这篇文章的讲解过程中，我们已经说出了它的主要用途。你发现没，使用动态代理我们居然可以在不改变源码的情况下，直接在方法中插入自定义逻辑。这有点不太符合我们的一条线走到底的编程逻辑，这种编程模型有一个专业名称叫 AOP。所谓的AOP，就像刀一样，抓住时机，趁机插入。

以上就是我理解的动态代理，要是有错误希望大家指出，不足之处望大家谅解，谢谢大家。
